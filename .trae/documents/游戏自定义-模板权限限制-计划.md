# 游戏自定义：模板权限限制（计划）

## Summary

为“游戏自定义”里的 **模板（template）** 增加可见性控制：

- 默认仅创建者可见（private）
- 在模板设置中开启“公开可见”后（public），其他用户可在“游戏自定义”的模板列表与“创建房间/首页选择模板”中看到并使用该模板
- 无论是否公开，模板的编辑/发布/回滚等写操作仅允许创建者

本计划以 **服务端强制校验** 为准，前端仅做体验优化（灰掉按钮/提示无权限）。

## Current State Analysis（基于仓库现状）

- 模板并非独立表；所有配置（rules/board/cards/template）统一存于 `AppData.configDocs`，持久化在 `apps/web/.data/neoblock.json`，解析入口在 [store.ts](file:///e:/code/ProjectNeoBlock/apps/web/lib/store.ts#L187-L227)。
- 当前 `ConfigDoc` 不包含 owner/visibility 等字段，[config.ts](file:///e:/code/ProjectNeoBlock/apps/web/lib/config.ts#L18-L28)。
- “游戏自定义”的模板列表来自 GET [api/config/list](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/list/route.ts)，未做鉴权/过滤（除系统默认模板隐藏）。
- 首页/创建房间的模板选择来自 GET [api/config/published](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/published/route.ts)，返回所有已发布模板（除系统默认模板隐藏），未做鉴权/过滤。
- 模板编辑页 [config/[docId]/page.tsx](file:///e:/code/ProjectNeoBlock/apps/web/app/config/%5BdocId%5D/page.tsx) 会直接请求 [api/config/get](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/get/route.ts) 获取 draft + published，并允许任何人调用 [api/config/draft](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/draft/route.ts) / publish / rollback 写入。
- 项目已有 NextAuth，会在 session 中提供 `session.user.id`（uid），见 [auth.ts](file:///e:/code/ProjectNeoBlock/apps/web/lib/auth.ts#L52-L58)，其他 API 已使用 `getServerSession(authOptions)` 获取 uid（例如 [room/create](file:///e:/code/ProjectNeoBlock/apps/web/app/api/room/create/route.ts#L46-L51)）。

## Decisions（已与需求对齐）

- 模板创建者 = **登录用户 uid**；未登录用户不允许创建/编辑模板（仍可浏览公开模板）。
- 存量模板（缺少 owner/visibility 字段）**默认视为公开可见**，以避免“突然消失”。
- “公开可见”打开后：公开模板会同时出现在
  - “游戏自定义”模板列表（给他人浏览/预览）
  - 首页/创建房间的“选择模板”列表（给他人使用）

## Proposed Changes

### 1) 数据模型：给 ConfigDoc 增加 owner 与可见性字段

修改 [config.ts](file:///e:/code/ProjectNeoBlock/apps/web/lib/config.ts)：

- 扩展 `ConfigDoc`：
  - `ownerId?: string | null`（模板创建者 uid；系统内置/seed 可为 null）
  - `visibility?: 'private' | 'public'`（仅对 kind === 'template' 有意义）
- 新增辅助类型/函数（放在 `lib/config-service.ts` 更合适）：
  - `effectiveTemplateVisibility(doc)`：`doc.visibility ?? 'public'`（兼容存量模板默认公开）
  - `canViewTemplate(doc, uid)`：
    - public：任何人可见（但建议仅对已发布模板对外展示）
    - private：仅 `uid === ownerId`
  - `canEditTemplate(doc, uid)`：仅 `uid && uid === ownerId`

修改 [store.ts](file:///e:/code/ProjectNeoBlock/apps/web/lib/store.ts) 的 `coerceConfigDoc`：

- 解析并保留 `ownerId` 与 `visibility`（不存在则保持 `undefined`，以触发“存量模板默认公开”的策略）

### 2) 创建模板：默认私有 + 写入 ownerId（且需要登录）

修改 [api/config/doc/route.ts](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/doc/route.ts)：

- 当 `kind === 'template'`：
  - 通过 `getServerSession(authOptions)` 获取 uid
  - uid 不存在 → 401 `UNAUTHORIZED`
  - 调用 `createNewDoc` 时传入 `ownerId: uid`，并设置 `visibility: 'private'`
- 其他 kind 保持现状（不引入额外权限变更）

修改 [config-service.ts](file:///e:/code/ProjectNeoBlock/apps/web/lib/config-service.ts)：

- 扩展 `createNewDoc` 入参以接收 `ownerId`（仅模板使用）
- 在创建 template doc 时写入 `ownerId` 与默认 `visibility: 'private'`

### 3) 模板“公开可见”设置：新增一个元数据更新接口

新增 API：`POST /api/config/visibility`

- 入参：`{ docId: string, visibility: 'private' | 'public' }`
- 规则：
  - doc.kind !== 'template' → 400
  - 未登录 → 401
  - 若 doc.ownerId 缺失（存量模板），首次写操作允许“认领”：
    - `doc.ownerId = uid`（仅当调用者为登录用户）
  - 非 owner → 403
  - 更新 `doc.visibility` 与 `doc.updatedAtMs`
- 返回：更新后的 `doc`（至少包含 `docId/kind/visibility/updatedAtMs`）

### 4) 读路径控制：列表/详情/发布列表按权限过滤（服务端强制）

修改 GET [api/config/list](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/list/route.ts)：

- 获取 uid（可为 null）
- 对模板（kind=template）按以下规则返回：
  - owner 的模板：全部返回（含 private、未发布）
  - 非 owner：仅返回 “公开可见 且 已发布” 的模板
  - 存量模板（visibility 缺失）：视为 public
  - 继续隐藏系统默认模板 docId（现有逻辑保留）
- 对 rules/board/cards：保持现状（不做权限变更）
- 建议在返回项中增加 `visibility`（仅模板）供前端标识 “公开/私有”

修改 GET [api/config/get](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/get/route.ts)：

- 获取 uid（可为 null）
- 若 doc.kind === 'template'：
  - owner：返回原结构（doc + draft + published + versions）
  - 非 owner：
    - 若模板不是 “公开且已发布” → 404（避免泄漏 docId 存在性）
    - 仅返回 `published`（draft=null；versions 过滤掉 draft/仅保留 published & archived）
    - doc 元信息中包含 `visibility`，以及 `canEdit: false`
- 对非 template：保持现状

修改 GET [api/config/published](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/published/route.ts)：

- 获取 uid（可为 null）
- templates 列表调整为：
  - 公开且已发布的模板（含存量模板视为 public）
  - 若 uid 存在：额外包含 “owner 的已发布模板”（即使 private，也让创建者能在创建房间时选到）
- defaultTemplateVersionId 逻辑保持现状

### 5) 写路径控制：保存草稿/发布/回滚仅允许创建者

修改以下接口：

- POST [api/config/draft](file:///e:/code/ProjectNeoBlock/apps/web/app/api/config/draft/route.ts)
- POST `api/config/publish`
- POST `api/config/rollback`

规则统一：

- 若目标 doc.kind !== 'template'：保持现状
- 若 doc.kind === 'template'：
  - 未登录 → 401
  - doc.ownerId 缺失（存量模板）：首次写操作“认领”为当前 uid
  - 非 owner → 403

### 6) 使用路径控制：创建房间/更新房间配置时禁止他人使用私有模板

修改：

- POST [api/room/create](file:///e:/code/ProjectNeoBlock/apps/web/app/api/room/create/route.ts)
- POST `api/room/config`

当请求中携带 `templateVersionId`：

- 解析模板时不仅要匹配 `publishedVersionId`，还需要校验该模板是否对当前请求者可用：
  - 未登录创建房间：仅允许 public 模板
  - 登录创建房间（account 模式）：允许 public 模板 + 自己的 private 模板

实现方式：

- 在 `lib/config-service.ts` 增加 `resolveTemplateDocByPublishedVersionId(data, versionId)`，返回 `ConfigDoc | null`
- 在 room 接口中拿到 doc 后再用 `canViewTemplate(doc, uid)` 进行判定（private 必须 owner）

### 7) 前端：模板设置增加“公开可见”开关（体验优化）

修改 [config/[docId]/page.tsx](file:///e:/code/ProjectNeoBlock/apps/web/app/config/%5BdocId%5D/page.tsx)：

- 扩展 `ConfigDocResponse` 类型，接收：
  - `doc.visibility?: 'private' | 'public'`
  - `canEdit?: boolean`（服务端返回；模板非 owner 为 false）
- 当 kind === 'template'：
  - 增加“模板设置”区块：
    - 开关/checkbox：`公开可见`
    - 触发 `POST /api/config/visibility`
  - 若 `canEdit === false`：
    - 隐藏或禁用 保存草稿/发布/回滚 等按钮
    - 表单输入设为只读（或禁用）
- 对 [config/page.tsx](file:///e:/code/ProjectNeoBlock/apps/web/app/config/page.tsx)：
  - “新建模板”在未登录时显示提示并禁用创建按钮（或保留按钮但展示 401 错误提示）
  - 模板列表项可显示标签：`公开`/`私有`

## Assumptions & Edge Cases

- 存量模板默认公开可见：这意味着系统升级后，未带 owner/visibility 的模板仍会被他人看到；当有人（登录用户）第一次对其执行写操作时，会被“认领”为该账号的模板并开始受私有/公开规则约束。
- 非 owner 访问公开模板详情页：
  - 只允许看已发布内容，避免泄漏草稿
  - 若公开模板未发布，则对他人不可见（列表与详情均不展示/404）

## Verification Steps（执行阶段完成后验证）

- 权限与可见性（两账号 A/B）：
  - A 创建模板（默认私有）→ B 在“游戏自定义”列表/创建房间均看不到
  - A 打开“公开可见”→ B 在“游戏自定义”列表可见、在创建房间可选中并成功创建房间
  - B 打开模板详情页只能看到已发布内容，保存/发布/回滚按钮不可用，接口返回 403
- API 回归：
  - 未登录 POST /api/config/doc(kind=template) 返回 401
  - 未登录或非 owner POST /api/config/draft/publish/rollback 返回 401/403
  - 未登录使用 private templateVersionId 创建房间返回 400/403（按实现选择的错误码）
